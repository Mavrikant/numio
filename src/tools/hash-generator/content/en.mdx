# Hash Generator

Compute a cryptographic hash of any text directly in your browser. Paste your input, pick an algorithm, and the digest appears instantly — in lowercase hexadecimal or Base64. Hashing uses the **Web Crypto API**, so nothing you type is ever sent to a server.

## What is a hash?

A hash function maps input of any size to a fixed-length **digest** — a kind of fingerprint. Good cryptographic hashes have three properties:

- **Deterministic** — the same input always yields the same digest.
- **One-way** — you cannot recover the input from the digest.
- **Collision-resistant** — it is infeasible to find two inputs with the same digest.

Hashes are used for file checksums, deduplication, digital signatures, password storage (with salting and a slow KDF), and verifying that data has not been tampered with.

## Choosing an algorithm

- **SHA-256** — the sensible default for almost everything. Fast, widely supported, 256-bit output.
- **SHA-384 / SHA-512** — larger digests with a higher security margin; SHA-512 can be faster on 64-bit hardware.
- **SHA-1** — produces a 160-bit digest but is **cryptographically broken**: practical collision attacks exist. Use it only for non-security checksums against accidental corruption, never to protect against an adversary.

## Hex vs Base64

The same digest bytes can be shown two ways. **Hex** is the most common (two characters per byte) and matches what tools like `sha256sum` print. **Base64** is more compact and convenient when embedding a digest in JSON or a URL.

## Privacy

Everything runs locally with `crypto.subtle.digest`. Your input never leaves your device.
